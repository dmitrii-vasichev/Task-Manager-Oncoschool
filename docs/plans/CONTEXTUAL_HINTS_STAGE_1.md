# Контекстные подсказки — Этап 1 (Kickoff)

Дата: 2026-02-24
Статус: In progress (draft for alignment)

## Цель этапа

Зафиксировать единый рабочий контракт перед внедрением UI-слоя подсказок: что показываем, где показываем, как измеряем эффект.

## Scope этапа 1

- Подготовить `Hint Inventory` для P0 экранов и сценариев.
- Утвердить единый шаблон текста и уровни приоритета.
- Утвердить технический контракт для frontend-реестра подсказок (route/role/state -> hint).
- Утвердить baseline-метрики и схему событий подсказок.

## Что не входит в этап 1

- Полная интеграция подсказок во все экраны.
- A/B тестирование формулировок.
- Отдельный Help Center.

## Рабочие артефакты этапа

- `docs/plans/CONTEXTUAL_HINTS_INVENTORY.md`
- Этот документ (`docs/plans/CONTEXTUAL_HINTS_STAGE_1.md`) как контрольный чеклист.

## Definition of Done для этапа 1

- Есть согласованный inventory по всем P0 зонам из спецификации.
- Для каждого P0 экрана есть минимум:
  - 1 `Orientation` или `State` hint;
  - 1 `Permission` hint;
  - 1 `Action-impact`/`Validation` hint.
- Есть утвержденные правила текста:
  - 1-2 коротких предложения;
  - структура `почему + следующее действие`;
  - timezone-указание для времени (МСК), где применимо.
- Есть согласованный минимальный telemetry-контракт (события + обязательные поля).
